Why Java on the Raspberry Pi?

Why Java on the Raspberry Pi? Why use Java - the server language - on such a small device? Well, actually, for me, there are many different reasons...

Raspberry Pi Header

The connection to electronic components

Why?

Because it's my language and want to learn more

First of all, it's my language. I have been mainly Java programming for over 10 years now and I feel most comfortable with it. Yes I've used C# and Python, Qt, JavaScript, but still I always return to Java as that allows me to do all the things I need to do the quickest way. I can imagine there is a lot of Java love in this room too, so why not extend your knowledge and enter the world of electronics

Why?

Because it's energy efficient

But a remarkable fact that most of us are not aware of, is that the amount of energy needed by a program, is highly different for the same kind of task for each programming language.

https://thenewstack.io/which-programming-languages-use-the-least-electricity/

This is scientific research, not some Java nerd coming up with some numbers... Compare Java with for instance Python...

Why?

Because of JavaFX

I know there are not a lot of us, but I do love JavaFX. Sorry, for the JavaScripties, but building an app with millions of dependencies and 16 different frameworks and tools is a nightmare...
There are numerous great libraries to build user interfaces, although I have to admit there are more for other languages and JavaFX has to catchup and needs a bigger user base... These are only a few, free, opensource.

Why?

Because of the libraries, community, and tools

And of course, it's not only the JavaFX libraries... It's all the others on top of that and the amazing tools, all provided by a very big and friendly community. Please look around, you're all part of it.
Vaadin mobile friendly web user interface, 100% written in Java. JBang: running Java code with dependencies like you are used to run bash scripts, without installing Java and Maven SDKMAN: switch between any Java version from all providers on all Linux and Mac platforms JavaFX: build desktop and mobile applications with 100% Java.

Why?

Because of the evolution with the 6-months release cycle

Each new release brings new features, cleaner code and better performance. Yes, also on the Raspberry Pi! Also JavaFX follows the same release cycle and each time includes improvements for embedded.

Why?

Focus on one thing to learn

This is probably the most important reason for us, people interested in technology, Java especially, and always want to learn more.

This project is intended to provide a friendly object-oriented I/O API and implementation libraries for Java programmers to access the full I/O capabilities of the Raspberry Pi platform. This project abstracts the low-level native integration and interrupt monitoring to enable Java programmers to focus on implementing their application business logic.

Minimal example

Blink a LED


    <dependency>
        <groupId>com.pi4j</groupId>
        <artifactId>pi4j-core</artifactId>
        <version>${pi4j.version}</version>
    </dependency>

    <dependency>
        <groupId>com.pi4j</groupId>
        <artifactId>pi4j-plugin-raspberrypi</artifactId>
        <version>${pi4j.version}</version>
    </dependency>
    <dependency>
        <groupId>com.pi4j</groupId>
        <artifactId>pi4j-plugin-pigpio</artifactId>
        <version>${pi4j.version}</version>
    </dependency>
						

var pi4j = Pi4J.newAutoContext();
						

private static final int PIN_LED = 22; // PIN 15 = BCM 22

var ledConfig = DigitalOutput.newConfigBuilder(pi4j)
      .id("led")
      .name("LED Flasher")
      .address(PIN_LED)
      .shutdown(DigitalState.LOW)
      .initial(DigitalState.LOW);

var led = pi4j.create(ledConfig);
						

while (true) {
      if (led.equals(DigitalState.HIGH)) {
           led.low();
      } else {
           led.high();
      }
      Thread.sleep(500);
}
